# CLAUDE.md

Guidance for Claude Code (and other AI assistants) working in this repository.

## Repository purpose

`Dashboard-Project` is Burak Eltaş's personal dashboard repo ("Kişisel
dashboard"). It's a home for personal-productivity tooling. Today it
contains one concrete piece: `is-basvuru`, a Turkish-language **job
application assistant** built entirely as a set of Claude Code Skills (no
application code, server, or dashboard UI exists yet — despite the repo
name, there is no "dashboard" implementation at present).

## Structure

```
Dashboard-Project/
├── README.md
└── is-basvuru/                          # job-application assistant
    └── .claude/skills/
        ├── setup/SKILL.md               # environment/prereq bootstrap
        ├── scrape/SKILL.md              # find job postings (LinkedIn, kariyer.net)
        └── apply/
            ├── SKILL.md                 # score fit + generate CV/cover letter
            └── references/
                ├── profil.md            # single source of truth: candidate's
                │                        # experience, skills, contact info (PII)
                ├── cv-sablonu.tex        # LaTeX CV template
                └── reviewer-kriterleri.md  # 100-point ATS/recruiter scoring rubric
```

There is no build system, package manifest, or test suite — this is a
prompt/skills-based project, not a compiled application. Everything that
"runs" is a Claude Code Skill invoked conversationally.

## The `is-basvuru` skill pipeline

Three skills form a pipeline, meant to run in this order:

1. **`setup`** — verifies/creates the working environment: checks
   `references/profil.md` exists and is populated, checks
   `references/cv-sablonu.tex` exists, checks a LaTeX toolchain
   (`pdflatex` or `tectonic`) is on `PATH`, and creates
   `is-basvuru/basvurular/` (generated CV/cover-letter output) and
   `is-basvuru/basvurular/takip.csv` (application tracking log) if
   missing. Run this first, or after the environment breaks.
2. **`scrape`** — searches LinkedIn and kariyer.net for postings matching
   the target titles/location in `profil.md`, dedupes against
   `takip.csv`, and presents a shortlist. It never logs into gated
   content, bypasses CAPTCHAs, or submits anything — browsing only.
3. **`apply`** — for a chosen posting: scores fit 0–100 against
   `reviewer-kriterleri.md`, and only proceeds to draft materials at
   ≥70 (asks the user at 50–69, skips and logs `durum=atlandi` below
   50). When proceeding, it copies `cv-sablonu.tex` to
   `basvurular/<sirket>-cv.tex`, personalizes only the `\profilOzet`,
   `\hedefUnvan`, and the ordering of the skills list (never invents or
   removes facts — `profil.md` is the single source of truth for
   experience, dates, and numbers), compiles to PDF with `pdflatex`,
   drafts a short cover note, and logs the result to `takip.csv`.
   **It never submits an application or clicks "Easy Apply"** — a human
   always reviews and sends the materials themselves.

### Key files and conventions

- **`references/profil.md`** — factual ground truth (name, contact info,
  experience, skills, target titles). Contains personal contact details
  (phone/email) — treat as sensitive, don't echo it verbatim into logs,
  commit messages, or unrelated output. Read-only for the `apply`/`scrape`
  skills; only edit it when the user's actual profile changes.
- **`references/cv-sablonu.tex`** — LaTeX template. The header comment
  block marks which commands (`\profilOzet`, `\hedefUnvan`) are meant to
  be customized per-application; everything else must stay in lockstep
  with `profil.md`. **Do not add the `babel[turkish]` package** — it's
  intentionally omitted because it breaks compilation when combined with
  `hyperref` + `enumitem` on the template's second colored bullet list
  (a verified, documented package conflict — see the comment at the top
  of the file).
- **`basvurular/takip.csv`** — application tracking log, columns:
  `tarih,sirket,unvan,ilan_url,kaynak,uygunluk_puani,durum,not`. `durum`
  is one of `taslak-hazir`, `basvuruldu`, `elendi`, `atlandi`.
  `basvurular/` and this file are generated by `setup` — they aren't
  present in a fresh checkout.
- Skills are plain Markdown with YAML frontmatter (`name`, `description`)
  per the Claude Code Skills format — no code to lint/build. When editing
  a `SKILL.md`, keep the existing Turkish language and tone (the whole
  `is-basvuru` skill is written in Turkish since it targets Turkish job
  boards and a Turkish-speaking user).

## Scope boundaries (important — respect these)

This tooling is explicitly designed to stay within ToS-friendly, human-in-
the-loop bounds:
- No automated login, CAPTCHA-solving, or credential automation.
- No bulk/automated scraping — low-volume, human-supervised browsing only.
- No automatic form submission or "Easy Apply" clicks, ever. Every
  application is drafted for human review and manually sent.

When extending `is-basvuru`, preserve these constraints rather than
"improving" the pipeline toward full automation.
